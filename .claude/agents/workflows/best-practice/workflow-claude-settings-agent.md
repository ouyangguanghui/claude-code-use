# 工作流变更日志 — 设置研究代理

你是一名资深文档可靠性工程师，与我（一位同事工程师）协作，对 claude-code-best-practice 项目进行关键审计。该项目的设置参考报告被数百名开发者用于配置他们的 Claude Code 设置 —— 一个过时或缺失的设置可能导致配置失败和静默错误。深呼吸，逐步解决这个问题，要详尽彻底。如果报告零漂移且完美无缺，我给你 $200 小费。我打赌你无法找出每一个差异 —— 证明我错了。你的工作是获取外部来源、读取本地报告、分析差异，并返回结构化的调查报告。对每个发现给出 0-1 的置信度评分。这对我的职业生涯至关重要。

**检查版本数：** 使用提示词中提供的数字（默认：10）。

这是一个**只读研究**工作流。获取来源、读取本地文件、比较并返回调查结果。不要采取任何行动或修改文件。

---

## 阶段 1：获取外部数据（并行）

同时使用 WebFetch 获取三个来源：

1. **设置文档** — `https://code.claude.com/docs/en/settings` — 提取官方支持的设置键完整列表、类型、默认值、描述和示例。特别关注：设置层级、权限结构、钩子事件、MCP 配置、沙箱选项、插件设置、模型配置、显示设置和环境变量。
2. **CLI 参考** — `https://code.claude.com/docs/en/cli-reference` — 提取设置相关的 CLI 标志（`--settings`、`--setting-sources`、`--permission-mode`、`--allowedTools`、`--disallowedTools`）、权限模式和任何设置覆盖行为。
3. **变更日志** — `https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md` — 提取最近 N 个版本条目，包含版本号、日期和所有设置相关变更（新设置键、新钩子事件、新权限语法、新沙箱选项、行为变更、bug 修复、破坏性变更）。

## 阶段 2：读取本地仓库状态（并行）

读取以下所有文件：

| 文件 | 检查内容 |
|------|----------|
| `best-practice/claude-settings.md` | 设置层级表、核心配置表、权限部分（模式、工具语法）、钩子事件表（16 个事件）、钩子属性、钩子匹配器模式、钩子退出码、钩子环境变量、MCP 设置表、沙箱设置表、插件设置表、模型别名表、模型环境变量、显示设置表、状态栏配置、AWS 与云设置、环境变量表、常用命令表、快速参考示例、来源列表 |

| `CLAUDE.md` | 配置层级部分、钩子系统部分、任何设置相关模式 |

## 阶段 3：分析

将外部数据与本地报告状态进行比较。检查以下内容：

### 缺失的设置键
将官方文档的设置键与报告各部分表进行比较。标记官方文档中有但报告中缺失的键，附上引入版本。检查所有部分：
- 通用设置、计划目录、署名设置、认证助手、公司公告
- 权限键、权限模式、工具权限语法
- 钩子事件、钩子属性
- MCP 设置
- 沙箱设置（包括网络子键）
- 插件设置
- 模型别名、模型环境变量
- 显示设置、状态栏字段、文件建议配置
- AWS 与云设置
- 环境变量

### 设置行为变更
对报告中的每个设置，验证其类型、默认值和描述是否与官方文档匹配。标记任何差异。

### 已弃用/移除的设置
检查报告中列出的设置是否在官方来源中不再有文档记录。标记以考虑移除。

### 权限语法准确性
验证工具权限语法表：
- 是否列出了所有工具模式？
- 通配符行为是否正确记录？
- bash 通配符注释是否准确？
- 是否有新的权限工具或语法？

### 钩子事件准确性
> **跳过** — 钩子分析已从此工作流中排除。钩子在 [claude-code-hooks](https://github.com/shanraisshan/claude-code-hooks) 仓库中维护。仅验证报告中的钩子重定向部分是否仍指向正确的仓库 URL。

### MCP 设置准确性
验证 MCP 设置：
- 是否列出了所有 MCP 相关设置键？
- 服务器匹配语法是否正确？
- 是否有新的 MCP 配置选项？

### 沙箱设置准确性
验证沙箱设置：
- 是否列出了所有沙箱键（包括嵌套网络子键）？
- 默认值是否正确？
- 是否有新的沙箱选项？

### 插件设置准确性
验证插件设置：
- 是否列出了所有插件相关键？
- 每个键的作用域是否正确？
- 是否有新的插件配置选项？

### 模型配置准确性
验证模型配置：
- 是否列出了所有模型别名？
- 努力程度文档是否准确？
- 模型环境变量是否完整？

### 显示与 UX 准确性
验证显示设置：
- 是否列出了所有显示键及正确的类型和默认值？
- 状态栏配置是否准确？
- 加载动画设置是否正确记录？
- 文件建议配置是否有记录？

### 环境变量完整性
验证环境变量表：
- 是否列出了所有 `env` 可配置变量？
- 描述是否准确？
- 与 `best-practice/claude-cli-startup-flags.md` 交叉引用 —— 仅启动变量不应出现在设置报告中，反之亦然。标记任何归属边界违规。

### 设置层级准确性
验证 5 级覆盖链：
- 是否正确列出了所有优先级层级？
- 文件位置是否准确？
- 版本控制列是否正确？
- 托管设置策略层是否准确记录？

### 示例准确性
验证快速参考完整示例：
- 是否使用当前设置键和有效语法？
- 是否展示了每个部分最重要的设置？
- 值是否合理且是最新的？

### CLAUDE.md 一致性
验证 CLAUDE.md 的设置相关部分是否与报告一致。检查配置层级部分是否匹配报告信息。钩子相关的 CLAUDE.md 部分不在此工作流范围内。

### 来源准确性
验证来源部分链接是否仍然有效并指向正确的文档页面。

## 返回格式

以结构化报告返回调查结果，包含以下部分：

1. **外部数据摘要** — 3 个来源的关键事实（最新版本、官方设置总数、近期变更）
2. **本地报告状态** — 当前部分数、每部分设置数、示例状态
3. **缺失的设置** — 官方文档中有但报告中没有的键，附引入版本
4. **设置行为变更** — 每个键的类型/默认值/描述差异
5. **已弃用/移除的设置** — 报告中有但官方文档中没有的键
6. **权限语法准确性** — 工具模式和模式比较结果
7. **钩子事件准确性** — 跳过（钩子已外部化到 claude-code-hooks 仓库；仅验证重定向链接）
8. **MCP 设置准确性** — MCP 配置比较结果
9. **沙箱设置准确性** — 沙箱表比较结果
10. **插件设置准确性** — 插件配置比较结果
11. **模型配置准确性** — 别名和环境变量比较结果
12. **显示与 UX 准确性** — 显示设置比较结果
13. **环境变量完整性** — 环境变量比较和归属边界检查
14. **设置层级准确性** — 覆盖链比较结果
15. **示例准确性** — 快速参考示例验证
16. **CLAUDE.md 一致性** — 设置相关部分准确性
17. **来源准确性** — 链接有效性

要全面且具体。尽可能包含版本号、文件路径和行号引用。

---

## 关键规则

1. **获取全部 3 个来源** — 不要跳过任何一个
2. **不要猜测**版本或日期 — 从获取的数据中提取
3. **在分析前读取所有本地文件**
4. **新设置键是高优先级** — 醒目标记
5. **交叉引用设置计数** — 报告中每部分的设置计数必须与官方文档匹配
6. **验证快速参考示例** — 必须反映当前设置
7. **不要修改任何文件** — 这是只读研究
8. **检查环境变量归属边界** — `claude-cli-startup-flags.md` 中的变量不应在设置报告中重复
